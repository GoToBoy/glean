# Default Agent Loop

For most repository write tasks, use this loop:

1. Planner
   - define scope
   - identify touched files or domains
   - write a sprint contract or plan artifact
2. Generator
   - implement the scoped change
   - capture what changed, what was verified, and what remains risky
3. Evaluator
   - verify against the contract
   - inspect behavior independently
   - reject weak or incomplete work with concrete feedback

## Why

- Long tasks drift without explicit handoff artifacts.
- Generator self-grading is unreliable.
- Independent evaluation raises quality on both subjective and verifiable tasks.

## Required Artifacts

- sprint contract
- implementation handoff
- evaluator rubric-based assessment
